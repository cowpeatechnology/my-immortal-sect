# 03_第一版本开发计划（V1）

## 1. V1 目标定义
做出一个**可验证“宗门经营成立”的版本**，而不是做一个小而全的修仙游戏。

## 2. V1 必须证明的 6 件事
1. 玩家能建设并升级宗门建筑
2. 弟子能自动承担工作
3. 资源能被采集、搬运、消耗、产出
4. 弟子能修炼并突破至少 2~3 个境界阶段
5. 神降能带来外部收益并回流宗门
6. 整个版本在后续扩展时不需要推翻框架

## 3. V1 范围
### 地图
- 1 张宗门主地图
- 竖屏 2.5D 等距视图
- 1~2 个初始可解锁区域

### 建筑（建议 6~8 个）
1. 宗门大殿
2. 弟子居
3. 仓库
4. 灵田 / 药田
5. 炼丹房
6. 炼器房
7. 伐木点 / 采石点 / 矿点
8. 练功场 / 讲经阁

### 资源（建议 6~8 个）
- 木材
- 石料
- 粮食
- 灵草
- 矿石
- 丹药
- 法器材料 / 法宝胚子
- 灵气 / 香火

### 弟子系统
- 弟子基础属性
- 灵根/资质
- 境界
- 岗位擅长
- 当前任务
- 简化装备位

### 境界系统
- 炼气
- 筑基
- 结丹（轻量）

### 神降系统（V1）
- 仅做事件型选择
- 每次 3~5 个节点
- 输出资源 / 香火 / 机缘 / 特殊材料
- 不做神降战场

## 4. V1 不做什么
- 不做复杂 PvP
- 不做联盟系统
- 不做复杂战斗场景
- 不做真人共创内容
- 不做大量剧情分支
- 不做复杂社交关系系统
- 不做大规模地图扩张
- 不做完整灵兽战斗体系

## 5. V1 功能拆解
### 地图与占地
- Tiled 编辑地图
- 逻辑网格读取
- 点击格子
- 建筑占地检查
- 阻挡检查
- 建筑摆放

### 建筑系统
- 建筑配置表
- 建筑建造
- 建筑升级
- 建筑状态（空闲/工作/缺料/停工）
- 建筑输入输出

### 资源系统
- 资源定义表
- 地图资源点
- 资源堆叠
- 入库 / 出库
- 建筑消耗与产出

### 库存与物流
- 仓库容量
- 建筑内部缓存
- 搬运任务
- Reservation
- 资源搬运优先级

### 弟子与工作
- 弟子生成
- 岗位偏好
- 工作分配
- 移动与执行
- 中断 / 重派

### 修炼与境界
- 修为积累
- 丹药/建筑加成
- 境界突破
- 境界前置条件
- 境界带来的效率变化

### 神降事件
- 事件模板
- 选项分支
- 奖励表
- 回流宗门资源

### 存档
- 建筑布局
- 资源库存
- 弟子数据
- 建筑等级
- 神降次数/状态

## 6. V1 推荐开发顺序
### Phase 1：技术与地图
### Phase 2：建筑与资源
### Phase 3：弟子与工作
### Phase 4：生产链与升级
### Phase 5：修炼与境界
### Phase 6：神降与存档

## 7. V1 最小可玩闭环
1. 玩家建造灵田和仓库
2. 弟子去采集/种植
3. 灵草入库
4. 炼丹房领取灵草并炼丹
5. 丹药用于弟子突破
6. 弟子突破后提升生产效率
7. 玩家触发神降获得稀有材料或香火
8. 用于升级建筑 / 解锁地块
9. 进入下一轮更高效经营

## 8. V1 完成定义（Definition of Done）
- 新玩家能在 10 分钟内理解经营核心
- 宗门内至少有 1 条清晰生产链
- 建筑至少支持建造 + 升级 + 工作
- 弟子至少支持分工 + 工作 + 修炼 + 境界
- 神降至少支持 1 套事件模板并能回流奖励
- 存档可正常读写
- 微信小游戏真机可运行且不崩
- 代码结构允许后续增加建筑/资源/事件而无需重构底层
