# 04_技术选型文档

## 1. 前端引擎
### 选型
**Cocos Creator**

### 选择原因
- 适合微信小游戏 / 抖音小游戏
- 适合 2D / 2.5D 竖屏项目
- 资源、场景、动画、UI、发布一体化
- 官方支持 TiledMap / Spine / DragonBones
- 适合轻中度经营类小游戏

## 2. 游戏逻辑框架
### 选型
**ESEngine**

### 选择原因
- TypeScript ECS 框架
- 可与 Cocos Creator 配合使用
- 适合把高频经营逻辑从 Node/Component 中抽出来
- 便于后续扩展 AI、网络、物理等模块

### 使用原则
- ESEngine 负责 simulation core
- Cocos 负责表现层与平台层
- 不把 UI 和全部场景逻辑都塞进 ECS

## 3. 地图系统
### 选型
**Tiled + Cocos TiledMap**

### 使用方式
- Tiled 负责静态地图与对象层编辑
- Cocos TiledMap 负责渲染
- 逻辑阻挡、占地、Reservation 由自定义网格维护

## 4. 角色动画系统
### 主选
**Spine**

### 备选
**DragonBones**

### 资产策略
- 只做男女两套基础骨架
- 统一动作库
- 外观通过 skin / attachment 组合
- 稀有服饰后置资源包按需加载

## 5. 寻路
### 推荐
**EasyStar.js**

### 后续扩展
如果后面需要更复杂的动态障碍和批量寻路，再考虑：
- 自写寻路缓存
- 热点路径缓存
- Worker 分担

## 6. 后端
### 首选
**Golang**

### 架构建议
V1 采用：
- Go API 服务
- MySQL/PostgreSQL
- Redis（可选）
- WebSocket（如有在线实时需求）
- 简单鉴权与存档

### 可选中后期方案
**Nakama + Go runtime**

## 7. 资源与包体策略
### 使用 Cocos Asset Bundle
拆分建议：
- 主包：登录、主界面、宗门主场景、基础角色
- Bundle_Events：神降事件
- Bundle_Characters：高级角色/服饰
- Bundle_Activities：限时活动

## 8. AI 辅助内容生产
### 可直接使用 AI 的部分
- 服饰概念图
- 武器/法宝设计图
- 神降事件文本
- 建筑设定图
- 文案模板
- UI 草图参考

### 不应直接完全交给 AI 的部分
- 最终可用 Spine 工程
- 最终地图逻辑设计
- 关键数值平衡
- 核心系统代码结构
