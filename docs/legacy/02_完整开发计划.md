# 02_完整开发计划（面向 Codex 的独立上下文文档）

## 1. 项目定义

### 1.1 产品一句话
这是一款运行在微信/抖音小游戏平台上的**2.5D 竖屏修仙宗门经营游戏**。玩家以“宗门意志/祖师”的上位视角经营一个独立宗门小世界，招募并培养弟子、建设建筑、安排生产、推动修炼和突破，并通过“神降”系统从外部世界带回资源与机缘，反哺宗门成长。

### 1.2 核心定位
- **主玩法**：宗门经营
- **副玩法**：神降事件
- **产品调性**：中式修仙、宗门建设、轻策略、强成长
- **平台形态**：微信 / 抖音小游戏，竖屏，轻中度在线

## 2. 顶层设计原则
1. 宗门经营必须是主干
2. 神降先做“外部输入口”
3. 先建立不会推翻的经营底盘
4. 玩法先轻后重

## 3. 产品核心循环

### 宗门内循环
1. 建筑建设 / 升级
2. 资源采集 / 生产 / 入库
3. 弟子接单 / 执行工作
4. 产出修炼资源 / 宗门资源
5. 弟子修炼 / 突破
6. 宗门规模提升 / 地块解锁

### 宗门外循环（神降）
1. 触发神降事件
2. 做出若干关键选择
3. 获得资源 / 香火 / 特殊道具 / 机缘
4. 回流宗门经营系统
5. 用于扩建、升级、培养弟子

## 4. 长期模块拆分
- 模块 A：经营基础层
- 模块 B：修仙增强层
- 模块 C：神降事件层
- 模块 D：在线与长期成长层
- 模块 E：共创扩展层（后期）

## 5. 全局里程碑
### 里程碑 0：技术原型
- Cocos + ESEngine + Tiled + Spine/DragonBones 基础链路
- 1 张小地图 + 1 个弟子 + 1 个建筑 + 简单寻路

### 里程碑 1：经营骨架原型
- 资源采集
- 入库
- 建筑建造/升级
- 弟子工作
- 简单境界成长

### 里程碑 2：V1 可玩版
- 6~8 个建筑
- 6~8 个资源
- 3~5 类弟子工作
- 1 条基础成长链
- 1 套神降事件入口
- 基础存档

### 里程碑 3：V1 测试版
- 数值平衡
- 新手流程
- 竖屏 UI 成型
- 首包与加载优化
- 微信小游戏真机运行

### 里程碑 4：V1.1 扩展版
- 更多神降事件
- 炼器 / 法宝增强
- 灵兽/养殖轻量接入
- 更多建筑与弟子标签

## 6. 开发顺序（推荐）
1. 技术底座
2. 地图与逻辑网格
3. 资源 / 建筑配置系统
4. 库存 / 物流 / Reservation
5. 弟子与工作流
6. 建筑升级
7. 修炼 / 境界
8. 神降事件系统
9. UI 与引导
10. 存档与数据持久化
11. 小游戏平台优化与发布
