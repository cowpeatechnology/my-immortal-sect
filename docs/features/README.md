# features/ 目录说明

本目录用于存放单功能或单专项的循环文档。

命名格式：

`F-xxx-<slug>.md`

示例：

- `F-001-storylet-dsl.md`
- `F-002-sect-map-playability-validation.md`
- `F-003-first-client-bootstrap.md`

每个 feature 文档都应基于：

- [feature-loop-template.md](/Users/mawei/MyWork/SlgGame/docs/templates/feature-loop-template.md)

它应记录：

- 问题定义
- 设计结论
- 验收标准
- 每一轮循环的变化
- 与既有功能或设计的冲突
- 当前状态

它不是 ADR，也不是项目总台账。
